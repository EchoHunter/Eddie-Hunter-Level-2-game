Kill a bunch of squids in a Dungeon

by: Eddie Hunter

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I felt a turn based game over a more arcade like game would be fun to create. I originally intended it to be turn based, but it changed overtime from me originally wanting it to be a bit like chess, there being 2 players and they control groups of characters, to it being a single playable character vs. many non playable character.

2. What I Learned

From user testing I discovered a bug where the game didn't end, and pretty quickly I figured it was a character getting stuck on the outside of the game area, which wasn't that hard to remove.

3. Architecture of My Game

My game is mostly just a class called GameManager, which has a bunch of methods in it which are then put in bigger methods which are then called by a class called GameRunner, and I have a bunch of other classes that make up the various objects like projectiles and the main enemies in the game.

4. Technical Implementation

A couple interesting things the game has is self guiding non playable characters, as well as click and drag movement for the player character. The self guiding NPCs are pretty simple, as all they do is move around the terrain towards the player character, and the click and drag movement is pretty simple, giving the player a radius they can move in and the player can drag the character the exact distance they want to move. I am pretty happy with these features however its hard to be proud of them myself because Daniel really helped with the features as I was struggling a lot with creating them.

5. Most Challenging Part

I think the most challenging part of this project was the NPCs moving around the terrain. I repeatedly wrote and deleted a lot of the code I wrote because it didn't work, and needed a lot of help with that particular feature. Some difficulties I faced were things like the system I made working for 1 piece of terrain or only 1 NPC, but not working with many terrain pieces or multiple NPCs, however after a few tries I was able to fix the problem. Later problems involved the NPCs moving slightly too far, and accidentally passing through or clipping terrain, however it was a pretty easy fix as I had to modify the method that moved the NPCs to account for longer movements.

6. Future Fixes and Updates

I would like to have more interesting terrain rather than just grey blobs, and more variety in the NPCs, even as simple as their animations being differently timed. If I were to continue working on the game, I would try to instead have various different shapes of terrain that I could give textures to, rather than just having various rectangles.
